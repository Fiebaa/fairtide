The Problem

A $5 coffee means nothing to one person and everything to another. Fixed pricing ignores this reality.

Fixed prices exclude people

A student in Lisbon and a banker in Zurich pay the same price. One can barely afford it, the other doesn't even notice. The gap between rich and poor keeps growing.

Fairtide closes the gap

One API call. You send the base price, the customer's income range, and their city. Fairtide returns a price that's fair — adjusted for what they can actually afford.

Revenue-Neutral by Design

"Won't I lose money?" — No. That's the whole point.

Fairtide tracks every transaction's delta — the difference between your base price and the fair price. When too many discounts accumulate, the algorithm automatically dampens future adjustments, nudging prices back toward your base price.

Over time, discounts for lower-income customers are offset by premiums from higher-income customers. Your revenue stays balanced.

Balanced — Revenue on track
Recovering — Dampening active, nudging toward base
Surplus — Premiums exceeded discounts, relaxing factors

Simple API Integration

One POST request. That's the entire integration.

Request
POST /v1/calculate
Authorization: Bearer your-api-key
Content-Type: application/json

{
  "basePrice": 4.50,
  "annualIncome": 28000,
  "locationId": "berlin-de"
}
Response
{
  "fairPrice": 3.78,
  "breakdown": {
    "basePrice": 4.50,
    "incomeFactor": 0.85,
    "locationFactor": 1.05,
    "adjustedIncomeFactor": 0.80,
    "fairPrice": 3.78
  },
  "balanceStatus": "balanced"
}
